# pnpm README Pitch Structure
- **Source URL**: https://github.com/pnpm/pnpm
- **Retrieved**: 2026-05-15

## Core Pitch Structure

The README opens with a direct value proposition: **"Fast, disk space efficient package manager"** followed by eight key selling points presented as bullet points:

- Performance ("Up to 2x faster than the alternatives")
- Storage efficiency (content-addressable system)
- Monorepo support
- Strict dependency access
- Deterministic lockfile system
- Node.js version management capability
- Cross-platform compatibility
- Production-tested credibility ("since 2016")

## Authority Positioning

The opening leverages third-party validation through a quote: "Microsoft uses pnpm in Rush repos with hundreds of projects and hundreds of PRs per day, and we've found it to be very fast and reliable." This establishes enterprise-scale credibility before discussing technical details.

## Installation & Accessibility

The README provides links to installation and usage documentation rather than inline commands, positioning itself as user-friendly while deferring complexity to dedicated guides. This approach emphasizes accessibility over technical depth in the opening section.

## Social Proof Indicators

Multiple engagement metrics appear prominently:
- Star count (35.1k)
- Sponsorship badges (Platinum/Gold/Silver tiers)
- Multilingual support links (Chinese, Japanese, Korean, Italian, Portuguese)
- Community channels (Discord, Twitter/X, Bluesky)

This layered structure builds trust through combination of speed metrics, institutional adoption, and community scale.
