# I installed NixOS on my daily driver, but I went back to Fedora in a week
- **Source URL**: https://www.xda-developers.com/i-installed-nixos-on-my-daily-driver-but-i-went-back-to-fedora-in-a-week/
- **Retrieved**: 2026-03-20
- **Type**: Blog post (XDA Developers)

## Author
Simon Batt

## Duration on NixOS
One week

## Primary Use Case
Daily driver laptop (replacing Fedora Kinoite)

## Key Reasons for Returning to Fedora

### Declarative System Philosophy
The fundamental issue was NixOS's declarative approach. Rather than installing applications through traditional methods, users must edit configuration files and rebuild the system. Batt describes encountering this immediately: "I had just slammed, face-first, into the world of declarative operating systems."

### Installation Complexity
Installing Google Chrome required editing a text configuration file rather than using a GUI app store. Batt notes: "Installing Google Chrome via a config file felt like programming—and it's not something I think I like."

### Learning Curve
NixOS "demands complex setup and ongoing learning," suggesting the barrier to entry was too steep for his workflow.

## Technical Details
The first barrier appeared within seconds of installation when Discover (KDE's app store) displayed an error message. Batt's immediate reaction—frantically searching for explanations on his phone—illustrates the friction point.

## Nuanced Take
Despite the frustration, Batt acknowledges appeal in the system's philosophy, describing NixOS as "the most enjoyable, customizable nightmare I've ever had," suggesting respect for its design even while recognizing it wasn't suitable for his needs.
