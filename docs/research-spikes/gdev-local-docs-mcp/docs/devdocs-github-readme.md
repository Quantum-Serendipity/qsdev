# DevDocs GitHub Repository - README

## 1. Architecture & How It Works

DevDocs operates as a **two-part system**: a Ruby scraper generates documentation and metadata, while a client-side JavaScript app powered by Sinatra handles the UI. The design strips HTML markup to prevent frame pollution and prefixes CSS classes with underscores to avoid conflicts.

## 2. Self-Hosting Instructions

**Manual installation requires:**
- Ruby 3.4.1
- libcurl
- JavaScript runtime (Node.js on Linux)

Process: Clone repository → install Bundler → run `bundle install` → download docs via `thor docs:download --default` → start server with `bundle exec rackup` → access at localhost:9292

## 3. Docker Support

Two pre-built images available:
- `ghcr.io/freecodecamp/devdocs:latest` (standard)
- `ghcr.io/freecodecamp/devdocs:latest-alpine` (smaller footprint)

Images update monthly automatically. Custom builds possible via included Dockerfile.

## 4. Documentation Count/List

The README states "multiple developer documentations" but doesn't specify exact count. Users can view available docs using `thor docs:list` command and download specific versions or all available documentation.

## 5. Scraper System

Two scraper types exist:
- **UrlScraper**: Downloads via HTTP
- **FileScraper**: Reads from local filesystem

Process includes link-following, HTML normalization, removing unnecessary content, fixing URLs, and building metadata indices using Nokogiri and HTML::Pipeline with Prism syntax highlighting.

## 6. API Endpoints & Programmatic Access

"The web app is all client-side JavaScript...files generated by the scraper." No REST API explicitly mentioned, but XHR loads content directly. Service workers and localStorage enable offline functionality.

## 7. Technology Stack

- **Backend**: Ruby, Sinatra, Sprockets
- **Frontend**: JavaScript (client-side)
- **Parsing**: Nokogiri, HTML::Pipeline
- **Highlighting**: Prism
- **Languages**: Ruby (80.2%), JavaScript (13.2%), SCSS (5.5%)

## 8. Documentation Storage Format

Documentation stored as:
- Normalized HTML partials
- Two JSON files: index + offline data
- JSON manifest containing metadata (name, version, update date)

## 9. Update Mechanism

"Currently no update mechanism other than `git pull origin main`" for code and `thor docs:download --installed` for documentation updates. Monthly Docker image updates included.

## 10. Security Considerations

- **License**: Mozilla Public License v2.0
- **Attribution requirement**: Documentation generated must credit DevDocs
- **Content source**: "All our content is pulled from third-party sources"
- No explicit security vulnerabilities mentioned in README
