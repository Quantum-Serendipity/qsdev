# Bun README Pitch Structure
- **Source URL**: https://github.com/oven-sh/bun
- **Retrieved**: 2026-05-15

## Tagline & Opening Pitch
Bun positions itself as an **"all-in-one toolkit for JavaScript and TypeScript apps"** that ships as a single executable. The opening emphasizes it's fundamentally a "fast JavaScript runtime designed as a drop-in replacement for Node.js."

## Key Selling Points (First Screenful)
1. **Speed & Efficiency**: Written in Zig and powered by JavaScriptCore, dramatically reducing startup times and memory usage
2. **Unified Tooling**: Replaces multiple tools — instead of "1,000 node_modules," you get one `bun` command covering runtime, testing, scripting, and package management
3. **Performance Advantage**: Explicitly contrasts itself as "significantly faster than existing options"

## Installation Presentation
The README offers multiple installation paths (curl script, npm, Homebrew, Docker), with the curl method marked as "recommended." It also highlights automatic canary builds and a simple upgrade command.

## Badge/Status Display
Badges showcase Discord community link, 90.6k GitHub stars, and a "speed = fast" indicator, visually reinforcing performance as the primary differentiator.

## Positioning vs. Alternatives
Rather than naming competitors directly, Bun emphasizes its "drop-in" compatibility with Node.js while stressing it "usable in existing Node.js projects with little to no changes" — suggesting ease of adoption over ecosystem disruption.
