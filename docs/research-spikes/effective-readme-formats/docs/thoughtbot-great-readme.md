# How To Write A Great README
- **Source**: https://thoughtbot.com/blog/how-to-write-a-great-readme
- **Retrieved**: 2026-05-15
- **Author**: Caleb Hearth (thoughtbot), originally published 2013, updated 2020

## Key Structural Recommendations

Hearth advocates for a README with these foundational elements:
- Clear description of what the project accomplishes
- Code examples demonstrating library usage
- Installation documentation
- Proper HTML formatting with an `<h1>` for the library name
- Two to three levels of headers for content organization

## Advice on Grabbing Attention

The author emphasizes that "Technical writing is still writing, and need not be dry and boring." He recommends using strong or emphasized text for concise descriptions and suggests incorporating relevant images — either depicting functionality or playing on wordplay in the project name.

## Code & Examples

Hearth stresses that developers understand code, noting that "syntax highlighted source are worth a thousand words." He highlights `plataformatec/simple_form` as exemplary for combining code samples with explanations, making it highly searchable and useful.

## Problem-Solving Approach

He advises documenting known issues directly in the README with solutions. His example: thoughtbot's `capybara-webkit` addresses persistent Qt dependency installation problems upfront.

## Additional Elements

Hearth recommends adding build badges from services like Travis CI and Code Climate, cautioning that failing builds can deter potential users.
