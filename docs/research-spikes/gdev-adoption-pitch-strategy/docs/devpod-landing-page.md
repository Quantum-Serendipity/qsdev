# DevPod Landing Page
- **Source URL**: https://devpod.sh
- **Retrieved**: 2026-05-15

## Hero Text & First Sentence
The primary heading positions DevPod as "Open Source Dev-Environments-As-Code." The page immediately contrasts it with GitHub Codespaces using "but..." to signal differentiation.

## Value Proposition
Three core pillars are highlighted:

1. **Open Source**: "No vendor lock-in. 100% free and open source built by developers for developers."

2. **Client-Only Architecture**: "No server side setup needed. Download the desktop app or the CLI to get started."

3. **Unopinionated Design**: "Repeatable dev environment for any infra, any IDE, and any programming language."

## Call to Action
Primary CTAs include "Download DevPod" buttons positioned above and below the fold, with platform-specific options (Mac Silicon/Intel, Windows, Linux, Other). Secondary CTAs offer sample projects across eight languages: Python, Node JS, Go, Rust, Java, PHP, C++, and Dotnet.

## Social Proof & Numbers
The page references 5.0k Slack community members. No customer logos or testimonials appear on this section, though a "Clone Dev's DevPod" testimonial section is mentioned but lacks substantive content.

## Competitive Positioning
DevPod directly challenges incumbent solutions by emphasizing infrastructure independence, absence of vendor lock-in, and no server-side overhead — positioning itself as the first tool for dev environments requiring "no heavyweight server-side setup."

## Visual Structure (Above the Fold)
Navigation bar, hero headline with "but..." positioning, three feature icons with descriptions, and prominent download buttons dominate the initial viewport. An illustration depicting DevPod's functionality anchors the lower fold section.
