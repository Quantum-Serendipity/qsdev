# GitHub README Best Practices: How to Write a README That Gets Stars

- **Retrieved**: 2026-05-15

## Core Strategies

Eight essential practices for writing high-impact READMEs:

**1. Clear Value Proposition**
The opening should immediately answer "What is this and why should I care?" in just two lines. Rather than vague descriptions, effective READMEs state concrete benefits.

**2. Visual Proof**
Demonstrations should appear immediately after the title, using GIFs (15-30 seconds maximum), screenshots, or architecture diagrams rather than relying solely on text descriptions.

**3. Strategic Badges**
Only include badges that build credibility — stars, license information, build status, and maintenance dates. Vanity metrics should be excluded.

**4. Rapid Onboarding**
The quick start section should allow users to get running in 30 seconds with copy-paste simplicity, requiring minimal setup steps.

**5. Scannable Feature Lists**
Tables format features more effectively than bullet points, enabling rapid comprehension of capabilities.

**6. Emotional Connection**
A dedicated section explaining the problem being solved and the motivation behind the project builds engagement.

**7. Documentation Architecture**
Rather than overwhelming the README, link to separate documentation files for getting started, API references, contribution guidelines, and changelogs.

**8. Welcoming Contribution Language**
Clear invitations to contributors, including "good first issue" labels, reduce barriers to participation.

## Recommended Structure

Logo and headline, visual demo, quick start, feature tables, problem explanation, documentation links, contribution guidance, and license information.

## Real-World Examples

Referenced successful projects include AFFiNE (60k+ stars), Supabase (70k+), and Excalidraw (80k+).

## Practical Guidelines

- **Optimal length**: 500-1,500 words
- **Focus platform coverage**: Include installation for the most common method
- **Image formats**: GIFs for demonstrations, PNGs for screenshots, SVGs for diagrams
- **Internationalization**: Provide multi-language versions for global reach
