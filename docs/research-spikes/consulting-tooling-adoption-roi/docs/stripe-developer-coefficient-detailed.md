# Stripe Developer Coefficient — Detailed Statistics

- **Source URL**: https://whoisnnamdi.com/leaving-software-on-the-table/ (analysis of Stripe study)
- **Original study**: https://stripe.com/files/reports/the-developer-coefficient.pdf
- **Retrieved**: 2026-03-20

## Developer Time Allocation (Weekly)

- **41.1 hours**: Typical developer work week
- **17.3 hours/week**: Total maintenance time (42% of work week)
  - **13.5 hours/week**: Addressing technical debt
  - **3.8 hours/week**: Fixing "bad code" (debugging, refactoring)
- **~23.8 hours/week**: Remaining for productive/strategic work

## Productivity Self-Assessment

- **68.4%**: Average self-rated team productivity (0-100% scale)
- **46% potential improvement**: Gap between current and optimal productivity
- **~19 additional hours weekly**: Equivalent productive development time possible

## Economic Impact

- **$600K**: Market value destroyed per engineer doing purely non-innovative work
- **$855K**: Market value added by average engineer balancing maintenance and innovation
- **$900B**: Aggregate global developer GDP (Stripe estimate)
- **$300B**: GDP shortfall from inefficiency (Stripe's original calculation)
- **~$425B**: Corrected GDP loss figure (using 46% productivity gap)
- **$670B**: Software output "left on the table" (developer shortage + productivity gaps combined)

## Developer Workforce

- **18-20M**: Global software engineering labor force
- **3M shortage**: Unfilled developer positions globally
- **1.4M**: Unfilled computer science jobs in US alone
- **15%**: Developer shortage percentage (3M/20M)

## Study Methodology

- Conducted by Harris Poll for Stripe, September 2018
- 1,000+ developers and 1,000+ C-level executives surveyed
- Multiple countries and industries
