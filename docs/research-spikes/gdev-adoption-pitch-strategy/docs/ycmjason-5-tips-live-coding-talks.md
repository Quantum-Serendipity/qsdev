# 5 Tips for Live-Coding Talks

## Tip #1: Genuine Enthusiasm
Authentic passion for your topic drives audience engagement. If you lack genuine enthusiasm, either select a different subject or work to convey excitement through delivery and word choices like "isn't it amazing/cool/awesome?"

## Tip #2: Delay the Talk to Build Comfort
Taking time to settle into the stage environment reduces nervousness and improves performance quality. Natural delaying tactics include:
- Taking selfies
- Playing relevant 30-second videos
- Chatting with attendees
- Visiting humorous websites briefly

This mirrors techniques learned from stage performance experience.

## Tip #3: Begin with Empty Project Creation
"Start the live-coding with `mkdir your-topic`" signals transparency to viewers. Building projects from scratch — through directory creation, git initialization, and npm setup — allows audiences to follow your complete workflow rather than joining mid-implementation.

## Tip #4: Minimize IDE Automation
Avoid IDE shortcuts that obscure processes from viewers. Warns against auto-generated files and code extraction features. If shortcuts are necessary, explicitly explain them first. Terminal commands should run manually for clarity.

## Tip #5: Prioritize Code Elegance
Clean, simple code supports comprehension. Specific recommendations include:
- Limiting indentation levels to four maximum
- Preferring modern loop syntax over traditional for-i patterns
- Using pure functions to minimize side effects
- Advocating contemporary language features
