# mise Landing Page
- **Source URL**: https://mise.jdx.dev
- **Retrieved**: 2026-05-15

## Hero Section
**Main Headline:** "Your dev env, _already prepped._"

**Subheading:** "One tool to manage languages, env vars, and tasks per project, reproducibly."

## First Sentence & Value Proposition
The opening establishes mise as a solution addressing developer environment setup. The core promise: "Dev tools, env vars, and tasks in one CLI"

The tool automates three key functions:
- Installing and activating project-specific tools with pinned versions
- Loading environment variables automatically
- Defining and executing project tasks

## Call to Action
Primary CTA: `curl https://mise.run | sh` (install command)
Secondary CTAs: "Getting Started" and "Demo" navigation links

## Conceptual Positioning
mise draws from the culinary term "mise en place" — the preparation ritual in professional kitchens. It positions itself as the "work before the work," handling prerequisites so developers can focus on coding.

**Above the Fold Content:**
- Logo and tagline
- Value proposition statement
- Live demonstration showing tool activation and task execution
- Primary installation command
- Navigation to documentation

## Social Proof Elements
- **Tool Support:** References 900+ tools in the pantry
- **Specific Examples:** Node, Python, Ruby, Go, Rust, Java, Terraform, Kubernetes
- **Related Project:** Mentions "aube," a Node.js package manager by the same creator

## Competitive Differentiation
Unified configuration through a single `mise.toml` file consolidates what typically requires multiple tools and configuration files.
