## Hero Section
**Main Headline:** "Bun is a fast JavaScript all-in-one toolkit"

**Subheading:** "Bun is a fast, incrementally adoptable all-in-one JavaScript, TypeScript & JSX toolkit. Use individual tools like bun test or bun install in Node.js projects, or adopt the complete stack with a fast JavaScript runtime, bundler, test runner, and package manager built in. Bun aims for 100% Node.js compatibility."

## Value Proposition
Bun offers four integrated tools:
1. **JavaScript Runtime** - Starts 3x faster than Node.js
2. **Package Manager** - 30x faster installs
3. **Test Runner** - Jest-compatible, replaces Vitest
4. **Bundler** - Replaces Vite and esbuild

## Key Performance Metrics
- **Bundling:** 269.1ms for 10,000 React components (vs 494.9ms Rolldown, 571.9ms esbuild)
- **HTTP Server:** 59,026 requests/sec (vs 25,335 Deno, 19,039 Node.js)
- **WebSocket:** 2,536,227 messages/sec (vs 1,320,525 Deno, 435,099 Node.js)
- **Database:** 28,571 queries/sec (vs 14,522 Node.js, 11,169 Deno)

## Feature Highlights
**Built-in APIs include:**
- PostgreSQL, MySQL, SQLite drivers
- S3 cloud storage
- Redis client with Pub/Sub
- WebSocket server
- Single-file executables
- YAML support
- Cookie parsing
- Password hashing (bcrypt, argon2)

## Design Goals
1. **Speed** - Fast startup and execution using JavaScriptCore engine
2. **Elegant APIs** - Minimal, optimized API set
3. **Cohesive DX** - Complete toolkit in one package

The page emphasizes that Bun works as a drop-in Node.js replacement while providing superior performance through tools like `Bun.serve()`, `Bun.sql`, and built-in bundling/testing capabilities.
