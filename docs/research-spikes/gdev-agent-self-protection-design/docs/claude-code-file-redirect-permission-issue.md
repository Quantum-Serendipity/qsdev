# Claude Code Issue #47701: Permission Walker Unhandled file_redirect Node Type

## Summary

The Bash permission evaluator in Claude Code uses a tree-sitter-bash AST walker to match commands against an allow-list. However, the walker lacks handlers for `file_redirect` nodes, causing it to log "Unhandled node type: file_redirect" and fall back to prompting for approval on every command containing shell redirection, pipes, or heredocs.

## The Core Problem

When the permission walker encounters a `file_redirect` node (generated by tree-sitter-bash for redirects like `>`, `>>`, `2>&1`, `<`, or heredocs `<<`), it has no handler. It logs "Unhandled node type: file_redirect" and falls through to require manual approval, even if the command is in the allow-list.

## Impact

The allow-list is effectively bypassed for any command containing:
- Shell redirection (`>`, `>>`, `<`)
- Pipes (`|`)
- Heredocs (`<<`)
- Stderr redirection (`2>&1`)

## Common Affected Patterns

1. Secret-redaction wrappers - pipe stdout through a sanitizer
2. Logging wrappers - tee stdout to a file
3. Exit-code preservation - `2>&1 | ... ${PIPESTATUS[0]}`
4. Heredoc multi-line commits - `git commit -m "$(cat <<'EOF' ... EOF)"`
5. Inline scripts - `node -e "..."` with string arguments

## Suggested Solutions

### Option 1 (Most Targeted)
Add handlers for: file_redirect, redirected_statement, pipeline, heredoc_body, command_substitution. When a redirect is encountered, unwrap to the inner command node and match against the allow-list.

### Option 2 (Most General)
Match the allow-list against the pre-hook command (original tool input) rather than the post-hook rewritten command.

## Key Insight for gdev

This confirms that Claude Code internally uses tree-sitter-bash for command parsing and permission evaluation. The fact that `file_redirect` nodes were unhandled means Claude Code's own permission system could not extract write targets from redirected commands. This validates the concern that reliable extraction of write targets from arbitrary bash is extremely difficult -- even Claude Code's own implementation has gaps.
