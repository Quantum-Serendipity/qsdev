# How to Create the Perfect README for Your Open Source Project
- **Source**: https://dev.to/github/how-to-create-the-perfect-readme-for-your-open-source-project-1k69
- **Retrieved**: 2026-05-15

## Introduction

The author explains that "Your README provides new users and potential contributors with the first impression of your project." She emphasizes that while a Contributing.md file enables pull request merges, "your README guides users on how to use your project."

## Recommended README Sections

### 1. Project Title
Start with a clear title. Open Sauced's GitHub repository cited as an example.

### 2. Description
Include "a few short statements describing why your project exists and what it does," with Slinkity's repository cited as a model.

### 3. Installation and Usage Instructions
Provide guidance on "how to install and use your project," referencing Timber's documentation as an exemplary approach.

### 4. Help/Support Information
"Let users know how they can get help," with RasaHQ's repository demonstrating this section effectively.

### 5. Documentation Links
"Include links to essential documentation," as shown in CodeNameOne's repository structure.

### 6. Contributors Section
"Show off your contributors," illustrated by the use-shopping-cart project's contributor display.

### 7. License Information
"Add a link to your project's license," with RasaHQ again serving as an example.

## Additional Enhancement Suggestions

The author recommends "going the extra mile" by including:
- Table of Contents (with auto-generation capability noted)
- Technology stack and feature lists
- Production instance links
- UI screenshots
- Dependencies
- Known issues

## Tools Mentioned

**Readme.so** is recommended as "an open source project built by Katherine Peterson" that "makes it easy to build and customize all the sections of your README." The Consumer Financial Protection Bureau's template is also suggested as a reference resource.
