# Time Tracking CLI Tools Survey

- **Source**: Multiple GitHub repositories and web searches
- **Retrieval Date**: 2026-05-14

## Toggl CLI Tools

1. **AuHau/toggl-cli (Python)** — PyPi-installable Python utility with CLI and API wrappers for Toggl API. Most established Toggl CLI. https://github.com/AuHau/toggl-cli
2. **watercooler-labs/toggl-cli (Rust)** — Unofficial CLI for Toggl Track written in Rust, using v9 API. https://github.com/watercooler-labs/toggl-cli
3. **joequery/Toggl-CLI (Python)** — Simple CLI for fast time tracking. https://github.com/joequery/Toggl-CLI
4. **tebeka/toggl (Go)** — Simple toggl command line client in Go. https://github.com/tebeka/toggl
5. **gabalese/toggl-cli** — Command line client for Toggl timekeeping. https://github.com/gabalese/toggl-cli

## Harvest CLI Tools

1. **zenhob/hcl** — Command-line tool for manipulating Harvest timesheets using Harvest API. Supports start, stop, resume, log time, view tasks. https://github.com/zenhob/hcl

## Clockify CLI Tools

1. **lucassabreu/clockify-cli (Go)** — Comprehensive CLI to manage time entries and projects on Clockify. Most established. https://github.com/lucassabreu/clockify-cli
2. **mentarch/clockify-cli** — CLI with features: fast performance, secure API key storage in OS keychain, rich reports (daily/weekly/monthly), project management, cross-platform. https://github.com/mentarch/clockify-cli
3. **BlythMeister/ClockifyCli** — Bridges Clockify, Jira, and Tempo. Intelligent time input, auto-sync Clockify to Tempo, Jira issue integration, timer start/stop. https://github.com/BlythMeister/ClockifyCli
4. **artefactual-labs/clockify-tool** — Python tool with YAML config. https://github.com/artefactual-labs/clockify-tool
5. **ABGEO/goclockify** — Terminal-based client for Clockify in Go. https://github.com/ABGEO/goclockify

## Assessment

- **Toggl**: Most CLI options, but mostly community-maintained, variable quality
- **Clockify**: Strong Go-based CLIs, good cross-platform support, free tier
- **Harvest**: Limited CLI ecosystem, mostly Ruby/older tooling
- All are third-party/community tools, not officially maintained by the time tracking vendors
- Quality varies significantly; most are single-maintainer projects
- None have the polish or reliability of, say, the GitHub CLI (gh)
