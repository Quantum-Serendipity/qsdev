# Progressive Disclosure — Nielsen Norman Group

- **Source URL**: https://www.nngroup.com/articles/progressive-disclosure/
- **Retrieved**: 2026-05-15

---

## Definition

Progressive disclosure is a UX strategy that reveals features in stages rather than all at once. It works by showing "only a few of the most important options" initially, then offering "a larger set of specialized options upon request."

## Core Principle

The pattern resolves a fundamental design tension: users want both powerful features AND simplicity. By deferring advanced options to secondary screens, designers can satisfy both needs without overwhelming novice users or slowing down experienced ones.

## Key Design Requirements

Two critical elements must work correctly:

1. **Right Feature Split**: Primary display must include frequently-needed features while secondary areas contain rarely-used options. The initial screen shouldn't contain confusing features that slow performance.

2. **Clear Progression Mechanics**: Users must easily understand how to access advanced options through obvious buttons or links with descriptive labels that set accurate expectations.

## Proven Benefits

Progressive disclosure improves three usability components:
- **Learnability**: Novice users focus on essential features
- **Efficiency**: Advanced users skip scanning unused options
- **Error reduction**: Hiding complexity prevents mistakes

Research indicates users actually develop better mental models when features are prioritized rather than presented comprehensively upfront.

## Implementation Considerations

- **Determine priority features** through task analysis, field studies, or usage analytics
- **Avoid excessive nesting**: Designs with 3+ disclosure levels typically fail; simplify if needed
- **Chunking**: Group advanced features logically using methods like card sorting

## Related Pattern: Staged Disclosure

Staged disclosure (wizards) differs by presenting features in task sequence order rather than importance hierarchy. It's effective for linear, independent steps but problematic when steps require frequent alternation.
