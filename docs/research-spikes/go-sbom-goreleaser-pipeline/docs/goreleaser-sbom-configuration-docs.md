# GoReleaser SBOM Configuration Documentation

## Overview

A Software Bill of Materials (SBOM) documents the components that comprise a software artifact. GoReleaser can generate one or more SBOMs for artifacts it produces.

## Configuration Structure

The SBOM pipeline analyzes artifacts and outputs SBOM files to the dist directory. By default, it creates an SBOM for each archive using Syft.

## Configuration Fields

### Core Settings

**id**
- Type: string
- Default: `'default'`
- Description: Unique identifier for the SBOM configuration

**cmd**
- Type: string
- Default: `'syft'`
- Description: Path to the SBOM generator command; process CWD is set to the dist directory

**documents**
- Type: array of strings
- Templates: Allowed
- Default varies by artifact type:
  - Binary: `["{{ .Binary }}_{{ .Version }}_{{ .Os }}_{{ .Arch }}.sbom.json"]`
  - Any: `[]`
  - Otherwise: `["{{ .ArtifactName }}.sbom.json"]`
- Description: Names of SBOM documents created (relative to dist dir); multiple values allowed only when artifacts is "any"

**args**
- Type: array of strings
- Templates: Allowed
- Default: `["$artifact", "--output", "spdx-json=$document", "--enrich", "all"]`
- Description: Command-line arguments passed to the SBOM generator

**env**
- Type: array of strings
- Default: `[ "SYFT_FILE_METADATA_CATALOGER_ENABLED=true" ]`
- Description: Environment variables passed to both the command and templates

**artifacts**
- Type: string enum
- Default: `'archive'`
- Valid options:
  - `any`: SBOM tool determines which artifacts to catalog
  - `source`: Source archive
  - `package`: Linux packages (deb, rpm, apk, etc)
  - `installer`: Installers (MSI, NSIS, macOS pkgs) [GoReleaser Pro]
  - `diskimage`: macOS DMG disk images [GoReleaser Pro]
  - `archive`: Archives from archive pipe
  - `binary`: Binaries from build stage

**ids**
- Type: array of strings
- Description: Artifact IDs to catalog; has no effect when artifacts is "source" or "any"

**disable**
- Type: boolean
- Default: `true`
- Templates: Allowed
- Note: Available since v2.10
- Description: Whether this SBOM configuration is disabled

## Template Variables

Available in template-supporting fields:

- `${artifact}`: Path to the artifact being cataloged (unavailable when artifacts is "any")
- `${artifactID}`: ID of the artifact being cataloged (unavailable when artifacts is "any")
- `${document}`: Generated SBOM filename; corresponds to `${document0}` when artifacts is "any"
- `${document#}`: Generated SBOM filenames by index (e.g., `${document0}`, `${document1}`)

## Example Configuration

```yaml
sboms:
  - id: foo
    documents:
      - "${artifact}.spdx.json"
    cmd: syft
    args: ["$artifact", "--output", "cyclonedx-json=$document"]
    env:
      - FOO=bar
      - HONK=honkhonk
    artifacts: archive
    ids:
      - foo
      - bar
    disable: true
```

## Limitations

"Container images generated by GoReleaser are not available to be cataloged by the SBOM tool."
