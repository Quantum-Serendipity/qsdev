# SLSA Provenance: Generate with GitHub Actions and Verify with slsa-verifier

## Core Concept

Provenance closes a critical gap left by SBOM and code signing alone. As illustrated by the SolarWinds SUNSPOT attack, "malware that lived on the build server, swapped the source code the moment a build started" can produce perfectly signed binaries with clean SBOMs. Provenance verifies "was this really built from the right source, on an unaltered builder."

## Key Terminology

**in-toto Statement**: A JSON structure declaring "for this subject (artifact), I am stating this predicate (claim)"

**DSSE (Dead Simple Signing Envelope)**: An envelope format where "the payload is base64-wrapped on the inside, and the signature wraps the outside"

**SLSA L3**: Generated by tamper-resistant build platforms where "user-defined build steps (data plane)" separate from "provenance generation logic (control plane)"

## Step 1: Verify a Public Artifact

### Required Tools
```bash
brew install slsa-verifier jq
slsa-verifier version
```

### Download Artifact and Provenance
```bash
mkdir slsa-hands-on && cd slsa-hands-on

curl -sLO https://github.com/slsa-framework/slsa-verifier/releases/download/v2.7.1/slsa-verifier-darwin-arm64

curl -sLO https://github.com/slsa-framework/slsa-verifier/releases/download/v2.7.1/slsa-verifier-darwin-arm64.intoto.jsonl
```

### Verification Command
```bash
slsa-verifier verify-artifact slsa-verifier-darwin-arm64 \
  --provenance-path slsa-verifier-darwin-arm64.intoto.jsonl \
  --source-uri github.com/slsa-framework/slsa-verifier \
  --source-tag v2.7.1
```

### Expected Output
```
Verified signature against tlog entry index 253498016 at URL: https://rekor.sigstore.dev/api/v1/log/entries/...
Verified build using builder "https://github.com/slsa-framework/slsa-github-generator/.github/workflows/builder_go_slsa3.yml@refs/tags/v2.0.0"
PASSED: SLSA verification passed
```


## Step 3: Inspect Provenance Content

Extract the DSSE envelope payload:
```bash
cat slsa-verifier-darwin-arm64.intoto.jsonl \
  | jq -r '.payload' \
  | base64 -d \
  | jq '{_type, predicateType, subject, predicate: {builder: .predicate.builder, buildType: .predicate.buildType, configSource: .predicate.invocation.configSource}}'
```

## Step 4: Generate Provenance from Your Own Repository

### Builder Configuration (.slsa-goreleaser.yml)
```yaml
version: 1

env:
  - GO111MODULE=on
  - CGO_ENABLED=0

flags:
  - -trimpath

ldflags:
  - "-X main.Version={{ .Env.VERSION }}"
  - "-s -w"

goos: linux
goarch: amd64

binary: hello-slsa-{{ .Os }}-{{ .Arch }}
```

### Release Workflow (.github/workflows/release.yml)
```yaml
name: Release with SLSA L3 Provenance

on:
  push:
    tags:
      - "v*"

permissions: read-all

jobs:
  build:
    permissions:
      id-token: write    # mint OIDC tokens
      contents: write    # upload Release assets
      actions: read      # read workflow metadata
    uses: slsa-framework/slsa-github-generator/.github/workflows/builder_go_slsa3.yml@v2.1.0
    with:
      go-version: "1.26"
      config-file: .slsa-goreleaser.yml
      evaluated-envs: "VERSION:${{ github.ref_name }}"
      upload-assets: true

  verify:
    needs: build
    runs-on: ubuntu-latest
    steps:
      - uses: slsa-framework/slsa-verifier/actions/installer@v2.7.1
      - uses: actions/download-artifact@v8
        with:
          name: ${{ needs.build.outputs.go-binary-name }}
      - uses: actions/download-artifact@v8
        with:
          name: ${{ needs.build.outputs.go-provenance-name }}
      - run: |
          slsa-verifier verify-artifact \
            "${{ needs.build.outputs.go-binary-name }}" \
            --provenance-path "${{ needs.build.outputs.go-provenance-name }}" \
            --source-uri "github.com/${{ github.repository }}" \
            --source-tag "${{ github.ref_name }}"
```

### Permission Requirements

- **id-token: write**: Enables GitHub to mint OIDC ID tokens, exchanged at Fulcio for short-lived certificates
- **contents: write**: Required for uploading Release assets
- **actions: read**: Needed to read workflow metadata (run id, sha, ref)

### Workflow Execution Flow

The build job compiles user code only; it never participates in provenance assembly or signing. Since builder_go_slsa3.yml runs as a separate workflow, no matter how dirty your user code is, it cannot reach the signing key. This structural separation satisfies SLSA L3.

## What the Verifier Actually Trusts

### Can Prove
- The binary at the given tag really came from the legitimate GitHub Actions builder
- Stops SUNSPOT-style attacks (tampering during build) and Codecov-style attacks (uploading without going through the build)

### Cannot Prove
- If the source itself contains malicious changes, provenance faithfully signs that input. That requires SLSA Source Track, four-eyes review, etc.

## Layering SBOM + Cosign + Provenance

| What It Proves | Tool | predicateType |
|---|---|---|
| Ingredients (contents) | Syft + cosign attest | `https://cyclonedx.org/bom` |
| Identity (author) | cosign sign | (signature itself, not a Statement) |
| Build path | slsa-github-generator | `https://slsa.dev/provenance/v1` |
