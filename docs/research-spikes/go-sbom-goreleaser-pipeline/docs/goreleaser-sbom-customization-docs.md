# GoReleaser SBOM Customization Documentation

- **Source URL**: https://goreleaser.com/customization/sbom/
- **Retrieved**: 2026-05-15

---

## Overview

GoReleaser enables generation of Software Bill of Materials (SBOMs) to describe components within software artifacts. The system can create multiple SBOM files for artifacts using configurable tools like Syft.

## Configuration Structure

The `sboms:` section in `.goreleaser.yaml` uses this format:

```yaml
sboms:
  - id: foo
    documents:
      - "${artifact}.spdx.json"
    cmd: syft
    args: ["$artifact", "--output", "spdx-json=$document"]
    env:
      - FOO=bar
    artifacts: archive
    ids:
      - foo
      - bar
    disable: false
```

## Key Configuration Options

**ID**: Unique identifier for each SBOM configuration (default: 'default')

**Documents**: Output filenames supporting templates. Defaults vary by artifact type:
- Binary: `{{ .Binary }}_{{ .Version }}_{{ .Os }}_{{ .Arch }}.sbom.json`
- Any: empty list
- Others: `{{ .ArtifactName }}.sbom.json`

**Command**: Path to SBOM generator tool (default: syft)

**Arguments**: Command-line flags with template support. Default uses `"spdx-json=$document"` output format.

**Environment Variables**: Passed to the SBOM command. Default includes `SYFT_FILE_METADATA_CATALOGER_ENABLED=true`

## Artifact Types

The `artifacts:` parameter accepts:
- `any`: Tool determines which artifacts to catalog
- `source`: Source archives
- `package`: Linux packages (deb, rpm, apk)
- `installer`: MSI, NSIS, macOS installers (Pro)
- `diskimage`: macOS DMG files (Pro)
- `archive`: Archives from build pipeline
- `binary`: Build stage binaries

## Available Template Variables

- `${artifact}`: Path to artifact being cataloged
- `${artifactID}`: Artifact identifier
- `${document}`: Generated SBOM filename
- `${document#}`: Specific SBOM file by index

## Limitations

Container images generated by GoReleaser are not available to be cataloged by the SBOM tool.
