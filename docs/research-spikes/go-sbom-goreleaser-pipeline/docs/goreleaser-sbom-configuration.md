# GoReleaser SBOM Configuration Documentation

## Overview

"A Software Bill of Materials (SBOM) is a description of the components that make up a software artifact." GoReleaser can generate one or more SBOMs for artifacts, with the default using Syft to create SBOM files in the dist directory.

## Default Configuration

When no custom SBOM configuration is specified:
- **Generator**: Syft
- **Default documents** vary by artifact type:
  - Binary: `{{ .Binary }}_{{ .Version }}_{{ .Os }}_{{ .Arch }}.sbom.json`
  - Any: (empty list)
  - Other: `{{ .ArtifactName }}.sbom.json`
- **Default arguments**: `["$artifact", "--output", "spdx-json=$document", "--enrich", "all"]`
- **Default environment**: `SYFT_FILE_METADATA_CATALOGER_ENABLED=true`

## Configuration Options

### Core Settings

| Option | Purpose | Default |
|--------|---------|---------|
| `id` | Unique identifier for the SBOM config | `'default'` |
| `cmd` | Path to the SBOM generator command | `'syft'` |
| `artifacts` | Which artifacts to catalog (any, source, package, installer, diskimage, archive, binary) | `'archive'` |
| `disable` | Whether to disable this configuration | `'true'` |

### Advanced Options

**documents**: List of output filenames (relative to dist directory). Supports templating with variables like `${artifact}`, `${document0}`, etc.

**args**: Command-line arguments passed to the SBOM tool. Templating is supported.

**env**: Environment variables passed to the SBOM command and templates.

**ids**: Specific artifact IDs to catalog (not applicable for source or "any" artifacts).

## Example Configurations

### SPDX JSON Format (Default)
```yaml
sboms:
  - id: spdx
    artifacts: archive
```

### CycloneDX JSON Format
```yaml
sboms:
  - id: cyclonedx
    cmd: syft
    args: ["$artifact", "--output", "cyclonedx-json=$document"]
    artifacts: archive
    documents:
      - "${artifact}.cyclonedx.json"
```

## Template Variables

Available in templating-enabled fields:
- `${artifact}`: Path to the artifact being cataloged
- `${artifactID}`: ID of the artifact
- `${document}`: Generated SBOM filename
- `${document#}`: Specific SBOM filename by index (e.g., `${document0}`)

## Important Limitations

"Container images generated by GoReleaser are not available to be cataloged by the SBOM tool." This means Docker/OCI images cannot be processed through the SBOM pipeline.
