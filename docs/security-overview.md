## Supply Chain Case Study: Trivy Compromise (March 2026)

In March 2026, attackers compromised the `trivy-action` GitHub Action by redirecting
76 of 77 version tags to malicious commits. The malicious code exfiltrated CI secrets
(environment variables, GITHUB_TOKEN, cloud credentials) to attacker-controlled
infrastructure. This affected thousands of repositories that pinned to version tags
rather than commit SHAs.

**Key lessons:**

1. **Pin CI actions to full commit SHAs**, not mutable version tags.
2. **Monitor CI egress traffic** — Harden-Runner's egress auditing would have detected
   the exfiltration immediately.
3. **Rotate secrets** after any suspected CI compromise.
4. **Vulnerability scanners are themselves attack surface** — scanning tools run with
   full access to your codebase and CI secrets.

This is why this profile includes harden-runner for CI egress monitoring.

## Tool Recommendations

### Socket.dev
Socket.dev provides behavioral analysis of packages, detecting supply chain attacks
that vulnerability scanners miss: typosquatting, install scripts, obfuscated code,
network access, and filesystem access patterns. Unlike scanners that rely on known
CVE databases, Socket proactively identifies suspicious behavior.

### OSV Scanner
Google's OSV Scanner queries the Open Source Vulnerabilities database, which
aggregates advisories from multiple sources (GitHub, NVD, language-specific databases).
It is free, open-source, and produces zero vendor lock-in.

### Harden-Runner
StepSecurity Harden-Runner provides egress traffic monitoring and policy enforcement
for GitHub Actions workflows. It detects compromised actions, credential exfiltration,
and unauthorized network access during CI builds.
