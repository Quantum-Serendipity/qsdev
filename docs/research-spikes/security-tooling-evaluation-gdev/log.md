- **Summary**: Prempti is a Falco-powered policy and visibility layer for AI coding agents (Rust, Apache-2.0, v0.3.0, created 2026-03-18, 42 stars). Intercepts every Claude Code tool call via PreToolUse hook, evaluates against Falco rule engine via Unix socket IPC, returns allow/deny/ask verdicts. Ships 58 default rules + 79 macros covering 7 security domains (boundary, sensitive paths, sandbox, threats, MCP/skill, persistence, self-protection). Runs as user-space daemon (Falco nodriver + plugin + supervisor). Fail-closed by design. Cross-platform (Linux/macOS/Windows). NOT a sandbox -- sees declared commands, not runtime side effects. Recommended as OPTIONAL configuration (`gdev enable prempti`) -- too heavy/immature for default but adds audit trail, ask verdicts, monitor mode, and MCP/self-protection rules gdev lacks. HIGHLY recommended as concept source: 6 patterns to borrow (self-protection rules, MCP poisoning detection, ask verdict, canonical path resolution, monitor mode, LLM-friendly output). Not recommended as default (infrastructure weight, 70% rule overlap with gdev, fail-closed risk, no Nix package, experimental status).

- **Summary**: Sense is a Go-native MCP server (88% Go, 688 commits, v0.84.3) that provides structural codebase understanding to AI agents through 4 tools: symbol graph navigation, hybrid semantic+keyword search, blast radius analysis (BFS with confidence decay), and convention detection (9 categories). Uses tree-sitter for 13-language parsing, bundled ONNX for embeddings, SQLite for storage. Integrates with Claude Code through 5 lifecycle hooks (session-start, pre-tool-use, post-tool-use, pre-compact, subagent-start). Critical finding: Sense is NOT a security tool — it solves AI agent navigation efficiency, not security. Benchmarks claim -47% tool calls, -32% tokens, -26% cost per task across 7 codebases. Maturity concerns: 4 stars, 1 fork, single author, O'Saasy license (MIT + SaaS restriction). Recommended as detect-and-offer config option in Phase 28 MCP registry — functionally replaces semble (Unit 11.3) with better capabilities and zero Python dependency. Also recommended as concept source for 5 patterns: silent hook failure, post-tool-use incremental re-indexing, pre-compact context injection, detect-and-nudge vs hard-block, idempotent setup with deep-merge.

## 2026-05-15 — Deep Dive: Cloudberry Automated Security Reviews
- **Type**: research
- **Status**: success
- **Depth**: deep
- **Sources**: [Automating Code Security Reviews](https://cloudberry.engineering/article/automating-code-security-reviews/) → `docs/cloudberry-automating-code-security-reviews.md`
- **Summary**: Cloudberry Engineering (originally published on Synthesia's blog) built a 6-phase AI security review pipeline: Prep→Map→Hunt→Dedup→Validate→Aggregate, using Semgrep for deterministic entry point detection, right-sized models per phase (Haiku for mapping, Opus for hunting/validation, Sonnet for dedup), and separated security context files. Key results: $3.88/review avg, 9.4 min, ~60% of raw findings discarded through dedup+validation. Assessed against gdev architecture: recommended as opt-in configuration option (`gdev enable deep-security-review`) plus 3 concept borrows: (1) `.security/` directory with architecture templates, (2) anti-false-positive patterns in security rules, (3) benchmarking protocol for review quality tracking. Not recommended as default (cost, Semgrep dependency, setup burden violate zero-prerequisites principle). Full analysis at `cloudberry-security-reviews-research.md`.
