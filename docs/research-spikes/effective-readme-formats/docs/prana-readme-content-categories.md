# Categorizing the Content of GitHub README Files (Prana et al.)
- **Source**: https://www.researchgate.net/publication/323302364_Categorizing_the_Content_of_GitHub_README_Files
- **Retrieved**: 2026-05-15 (abstract and findings from search results; full paper behind paywall)
- **Authors**: Prana et al. (2019)

## Study Overview

Analyzed 4,226 README sections from 393 GitHub repositories. Proposed a content taxonomy and built an automated classifier for README content.

## Seven Content Categories Identified

Based on the study's analysis, README sections were classified into these categories:

1. **What** — What the project is (description, purpose)
2. **Why** — Motivation for the project
3. **How** — How to use it (usage instructions, examples)
4. **When** — Version/status information, changelogs
5. **Who** — Contributors, maintainers, acknowledgments
6. **References** — Links to external resources, related projects
7. **Contribution** — How to contribute to the project

## Key Findings

- Many projects lack basic information about **purpose and status** despite having rich usage instructions
- There is a gap between what developers need (understanding what a project is and why it exists) and what maintainers tend to write (detailed how-to instructions)
- The "What" and "Why" categories are frequently underrepresented relative to their importance for project discovery and evaluation
- Projects with more complete coverage across all seven categories tended to be better received

## Significance

This taxonomy became foundational for subsequent research on README quality, including the Venigalla & Chimalakonda (2022) and Wang et al. (2023) studies that used these categories to study popularity correlations.
