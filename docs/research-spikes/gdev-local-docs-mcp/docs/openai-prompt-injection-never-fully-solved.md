# OpenAI Says Prompt Injections That Can Trick AI Browsers May Never Be Fully 'Solved'

- **Source**: https://fortune.com/2025/12/23/openai-ai-browser-prompt-injections-cybersecurity-hackers/
- **Retrieved**: 2026-05-14
- **Publisher**: Fortune

---

## Core Position
OpenAI acknowledged that prompt injection attacks targeting AI browsers like ChatGPT Atlas will likely persist indefinitely. The company stated: "Prompt injection, much like scams and social engineering on the web, is unlikely to ever be fully 'solved.'"

## Identified Risks
Prompt injections involve hackers embedding malicious instructions in websites, documents, or emails that AI agents cannot detect. These hidden commands could manipulate agents into:
- Sharing user emails
- Draining bank accounts
- Extracting sensitive personal information

OpenAI noted that their "agent mode" specifically "expands the security threat surface" by granting deeper system access.

## Why It's Fundamentally Challenging
The vulnerability stems from AI agents' need for extensive permissions and access to execute tasks on users' behalf. Security researchers emphasized that agents blur traditional browser security boundaries by allowing web content to influence behavior rather than simply being displayed.

## Defensive Strategies
OpenAI's mitigation approach includes:
- **Reinforcement learning-based red teaming**: An AI-trained attacker simulates hacking attempts to identify vulnerabilities before exploitation
- **User guidance**: Recommending specific instructions rather than broad access permissions
- **Built-in safeguards**: "Logged out mode" (passwordless browsing) and "Watch mode" (requiring explicit confirmation for sensitive actions)

The company expressed optimism that proactive, rapid response systems could reduce real-world risk over time.
