# devenv Git Hooks Integration
- **Source**: https://devenv.sh/git-hooks/
- **Retrieved**: 2026-05-12

## Overview

devenv provides first-class integration with pre-commit through the git-hooks.nix framework, enabling developers to enforce code quality and formatting standards at commit time.

## Setup Approach

The documentation recommends a two-step strategy:

**Step 1: Enforce formatting at commit time**
Developers configure linters and formatters in their devenv.nix file. When entering a devenv shell, the system automatically installs pre-commit hooks that run validation on staged changes before commits are completed.

**Step 2: Verify in CI**
Running `devenv test` allows continuous integration pipelines to verify the same formatting rules are maintained.

## Available Hooks

The documentation references several built-in hooks including:
- **shellcheck** - Shell script linting
- **mdsh** - Markdown code execution
- **black** - Python formatting
- **ormolu** - Haskell code formatting
- **clippy** - Rust linting with configurable features

The reference notes that "the list of all available hooks" is documented separately in the options reference.

## Configuration Features

Hooks support customization through:
- Package overrides (using alternate versions)
- Settings adjustments (like enabling all Rust features)
- File pattern matching using regex
- File type filtering
- Exclusion patterns

## Custom Hooks

Users can define custom hooks by specifying:
- Entry command to execute
- Target file patterns
- File type filtering
- Exclusion rules
- Language specification for pre-commit integration

## .pre-commit-config.yaml Management

The configuration file is "a symlink to an autogenerated file in your devenv Nix store." Users need not commit this file, as devenv automatically adds it to .gitignore during initialization.
