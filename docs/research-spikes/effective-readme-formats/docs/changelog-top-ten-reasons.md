# Top Ten Reasons Why I Won't Use Your Open Source Project
- **Source**: https://changelog.com/posts/top-ten-reasons-why-i-wont-use-your-open-source-project
- **Retrieved**: 2026-05-15
- **Author**: Adam Stacoviak

## The Ten Reasons

1. **No README** — Projects lack clear documentation at the entry point
2. **Missing Tests/Specs** — Absence of testing infrastructure and examples
3. **No Project Home Page** — Relying solely on GitHub without dedicated branding
4. **Design Issues** — Poorly styled presentation lacking professional appearance
5. **No Domain Name** — Missing custom domain for improved discoverability
6. **No Twitter Account** — Absent dedicated social media presence
7. **Unclear Licensing** — Vague or missing legal terms for usage
8. **No Community Outreach** — Failing to engage with relevant audiences
9. **No Conference Presence** — Lack of public speaking or community visibility
10. **Not Submitted to Changelog News** — Missing promotion through major channels

## Specific README Guidance

The article emphasizes that "the Readme is often the first look someone has at your project." Effective READMEs should include:

- Clear project description
- Installation instructions
- Help resources (documentation, mailing lists)
- Contribution guidelines
- Contributor attribution
- Credits and inspiration sources

## Documentation Emphasis

Strong testing documentation matters significantly. "A project without tests is harder to trust, contribute to, and looks more like a hobbyish hack." The piece advocates for "scenario-based code" examples demonstrating practical usage rather than requiring external research.
