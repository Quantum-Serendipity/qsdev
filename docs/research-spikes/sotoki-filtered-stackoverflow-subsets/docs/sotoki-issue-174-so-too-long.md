# Stack Overflow ZIM Build Performance Issue #174

## Summary
This GitHub issue documents a performance problem with the sotoki project, which creates offline ZIM archives of Stack Overflow content.

## Key Details

**Main Problem:**
The issue states that "stackoverflow's duration is not manageable" and calls for investigation into the performance bottlenecks causing extended build times.

**Technical Context:**
The issue mentions that a previous contributor (@dattaz) had required BTRFS (a Linux filesystem) for disk operations, but this requirement was not currently being implemented. The opening poster questions whether this filesystem choice was performance-related.
