# SLSA Security Levels (Build Track)

SLSA organizes security into tracks, with the Build track being the primary one. Four levels are defined (L0-L3).

## Build L0: No Guarantees
- No requirements; represents absence of SLSA compliance
- Intended for development/test builds on a single machine

## Build L1: Provenance Exists
- Package has provenance showing how it was built
- Can be used to prevent mistakes but is trivial to bypass or forge
- Requires: Consistent build processes, provenance generation showing builder identity, build process, and inputs
- Benefits include easier debugging and mistake prevention during releases

## Build L2: Hosted Build Platform
- Forging the provenance or evading verification requires an explicit attack, though this may be easy to perform
- Requires: All L1 requirements plus signed provenance generated by a hosted platform
- Adds tampering prevention through digital signatures

## Build L3: Hardened Builds
- Forging the provenance requires exploiting a vulnerability beyond most adversaries' capabilities
- Requires: All L2 requirements plus strong controls preventing:
  - Cross-run influence
  - Secret/credential access to user-defined build steps
- Addresses insider threats and compromised credentials
