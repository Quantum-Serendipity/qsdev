# Trivy SBOM Scanning Capabilities

## Supported Input Formats

Trivy accepts multiple SBOM formats for vulnerability and license scanning:

- **CycloneDX** (JSON format; XML not currently supported)
- **SPDX** (tag-value and JSON formats)
- **CycloneDX-type attestations**
- **KBOM** (Kubernetes Bill of Materials in CycloneDX format)

The tool automatically detects format when scanning SBOM files.

## Basic Usage

```
trivy sbom /path/to/sbom_file
```

By default, vulnerability scanning runs automatically. Users can customize scanner selection using `--scanners vuln,license` or `--scanners license` alone.

## Accuracy Considerations

**Critical limitation**: "Passing SBOMs generated by tool other than Trivy may result in inaccurate detection because Trivy relies on custom properties in SBOM for accurate scanning."

This dependency on Trivy-specific metadata means third-party SBOMs may not scan reliably.

## KBOM and Kubernetes Support

For Kubernetes distributions, KBOM scanning supports:
- Kubernetes upstream
- Rancher RKE2

Users generate KBOMs via `trivy k8s --format cyclonedx cluster`, then scan them as standard SBOM files.

## VEX Support

Trivy references VEX (Vulnerability Exploitability Exchange) capabilities in its documentation. Supports VEX Hub for suppressing non-applicable vulnerabilities.
